Data Index

Scaffolding out a 'data index' page. Here's what this does:

  • Ingests Indicators.json
  • Loops through that json to create a table of Indicators, Measurement types, Geographies, and times

A few notes:

  • For time, it grabs the final value from AvailableTimes, to give us "most recent year." But, this is presented as a string, so the sort is a little bit off.
  • Just the list of geographies is also awkward. Wonder if there's a better way to do this. We could do "finest geography" but will that be helpful?
  • This lets us scan measurement types and see things that can be improved upon (labels), like instances where age is in "measurement type."
